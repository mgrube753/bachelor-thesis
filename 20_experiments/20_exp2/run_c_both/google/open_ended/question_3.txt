Frage: Ein Entwicklerteam arbeitet an einem neuen drahtlosen Kommunikationssystem für vernetzte Sensoren in einer Smart-City-Anwendung, das auf einem gemeinsamen Übertragungsmedium basiert. **Veranschaulichen** Sie, wie der Data Link Layer in diesem System die im Text beschriebenen Kernaufgaben und Mechanismen **anwenden** müsste, um sowohl eine zuverlässige, scheinbar fehlerfreie Datenübertragung zwischen den Sensoren und einem zentralen Gateway zu gewährleisten als auch den effizienten Zugriff auf das geteilte drahtlose Medium zu **lösen**.

Antwort: ~In diesem Szenario müsste der Data Link Layer des drahtlosen Kommunikationssystems mehrere Mechanismen anwenden:
1.  **Fehlerbehandlung:** Er würde die Eingangsdaten von den Sensoren in Datenframes aufteilen und diese sequenziell übertragen. Um die Leitung "fehlerfrei" erscheinen zu lassen, würde er einen Mechanismus zur Fehlererkennung und -korrektur (z.B. CRC-Prüfsummen) sowie, bei Bedarf, Bestätigungs-Frames (Acknowledgements) vom Gateway erwarten. Sollte ein Frame nicht korrekt empfangen werden, würde der Data Link Layer eine Wiederholung des Frames anfordern, um die vom Netzwerk-Layer erwartete Zuverlässigkeit zu gewährleisten.
2.  **Flusskontrolle:** Da Sensoren möglicherweise nur sporadisch Daten senden und das Gateway Engpässe aufweisen könnte, müsste der Data Link Layer einen Flusskontrollmechanismus implementieren. Dieser würde sicherstellen, dass ein schnell sendender Sensor (Transmitter) das langsamere Gateway (Receiver) nicht überlastet. Dies könnte durch Pufferverwaltung am Gateway und Rückmeldungen an den Sensor über die aktuelle Empfangsbereitschaft erfolgen, sodass der Sensor seine Sendegeschwindigkeit anpassen kann.
3.  **Medium Access Control (MAC):** Für den effizienten Zugriff auf das geteilte drahtlose Medium in einer Broadcast-Umgebung ist der MAC-Sublayer entscheidend. Er würde Regeln definieren, wie und wann die Sensoren auf den Kanal zugreifen dürfen, um Kollisionen (gleichzeitiges Senden mehrerer Sensoren) zu vermeiden oder zu minimieren. Beispiele für anzuwendende MAC-Mechanismen wären CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance), bei dem ein Sensor vor dem Senden prüft, ob der Kanal frei ist, oder Time Division Multiple Access (TDMA), bei dem jedem Sensor feste Zeitschlitze für die Übertragung zugewiesen werden. So wird ein fairer und geordneter Zugriff aller Sensoren auf das Medium ermöglicht.
